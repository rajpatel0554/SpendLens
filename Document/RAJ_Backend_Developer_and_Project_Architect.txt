RAJ — Backend Developer & Project Architect
Responsible for: API, Database, Auth, Deployment, ML Integration
Tools to use: Python 3.11, FastAPI, SQLite → PostgreSQL, JWT, pdfplumber, pandas, Render.com, GitHub, Postman
Phase 1 — Project Setup (Days 1–3)
Task 1 — Initialize GitHub Repository Create the main project repo. Make these folders inside: /backend, /frontend, /ml, /data. Write a basic README file explaining what SpendLens is. Set up .gitignore so Python cache files don't get pushed. Create two branches: main (final working code only) and dev (everyone works here).
Task 2 — FastAPI Project Skeleton Install FastAPI and uvicorn. Create main.py with one health check endpoint (GET / returns {"status": "ok"}). Set up CORS so Pratha's React frontend can call your API without browser errors.
Task 3 — Database Schema Design Design 4 tables using SQLAlchemy ORM:
users — id, email, password_hash, created_at
transactions — id, user_id, date, description, amount, category, is_anomaly
categories — id, name
anomalies — id, transaction_id, reason, flagged_at
Write migration scripts so the database can be recreated from scratch anytime.
Task 4 — JWT Authentication Build /register and /login endpoints. Hash passwords using bcrypt before saving. On login, return a JWT token. Add a middleware decorator that protects all other routes — if no valid token is sent, return 401 Unauthorized.

Phase 2 — Core API Endpoints (Days 4–8)
Task 5 — File Upload Endpoint Build POST /upload. Accept CSV or PDF file. For PDF, use pdfplumber to extract text. Pass the raw file to Vaidehi's parser function. Take the clean rows she returns and save them to the database. Return parsed rows as JSON to the frontend.
Task 6 — Transaction CRUD Endpoints
GET /transactions — with filters: date range, category, amount range. Return paginated JSON (20 rows per page).
POST /transactions/manual — let users add a transaction by hand.
DELETE /transactions/:id — remove a transaction.
Task 7 — ML Model Integration When the server starts, load Manan's model.pkl and vectorizer.pkl files into memory. Build POST /predict — takes a transaction description string, returns predicted category. On every upload, run batch prediction on all new transactions automatically.
Task 8 — Analytics Endpoints
GET /analytics/summary — returns total spend, breakdown by category, month-over-month comparison. Compute using pandas.
GET /analytics/anomalies — returns all transactions flagged as anomalies by Manan's detection function.
Phase 3 — Reports, Testing & Deployment (Days 9–18)
Task 9 — PDF Report Generator Build GET /report/monthly. Use fpdf2 library. Pull this month's data from the database. Format into a PDF with: summary table, top 3 spending categories, list of anomalies, one AI-written insight line. Return as a downloadable PDF file.
Task 10 — API Testing with Postman Create a Postman collection covering every single endpoint. Test edge cases: empty CSV upload, expired JWT token, duplicate transaction upload, invalid date format. Export the collection and share it with Sarthak for QA.
Task 11 — Deploy to Render.com Create a free Render.com account. Deploy the FastAPI app as a web service. Set environment variables: DATABASE_URL, JWT_SECRET, ALLOWED_ORIGINS. Connect your GitHub repo for auto-deploy on every push to main. Test every endpoint on the live URL.
