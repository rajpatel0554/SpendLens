TANVI — SRS Writer & Documentation Specialist
Responsible for: Software Requirements Specification, use cases, data flow diagrams, final project report, demo script
Tools to use: Google Docs, draw.io (free at app.diagrams.net), MS Word
Phase 1 — Requirements Document (Days 1–3)
Task 1 — Write the SRS Document Create a Google Doc titled "SpendLens — Software Requirements Specification". Include these sections:
Introduction — What is SpendLens? What problem does it solve?
Project Scope — What is included, what is not included
Functional Requirements — List every feature the app must have (upload CSV, auto-categorize, show dashboard, detect anomalies, download report)
Non-Functional Requirements — App must load in under 3 seconds, passwords must be hashed, JWT tokens must expire after 24 hours
Constraints — Must be free to host, must work on mobile browser
Task 2 — User Stories Write at least 10 user stories in this exact format: "As a [user type], I want to [do something], so that [I get this benefit]."
Examples:
"As a user, I want to upload my bank CSV, so that I can see my spending automatically."
"As a user, I want to be alerted when my spending is unusually high, so that I can control my budget."
Task 3 — Data Flow Diagram Go to app.diagrams.net (free). Draw a diagram showing how data moves through the system: User uploads file → Backend receives → Vaidehi's parser cleans → Manan's ML categorizes → Database stores → Dashboard displays → Report downloads. Export as PNG and attach to the SRS doc.
Phase 2 — Support the Team (Days 4–12)
Task 4 — Use Case Document For each of the 5 main features, write a full use case:
Actor (who does it)
Precondition (what must be true before)
Main Flow (step by step what happens)
Alternate Flow (what if something goes wrong)
Postcondition (what is true after it succeeds)
Task 5 — Keep SRS Updated If any team member builds something differently from the SRS, update the document immediately. You are the official record of what the app does. If there is ever a disagreement about a feature, your SRS document is the final answer.
Phase 3 — Final Documentation (Days 13–17)
Task 6 — Final Project Report Write a 10–15 page report in Google Docs. Sections:
Abstract (half page summary)
Introduction (problem statement)
Literature Review (mention 2–3 similar apps like Mint, Walnut, ET Money)
System Design (architecture diagram, database schema)
Implementation (describe what each team member built)
Testing Results (use Vaidehi's ML accuracy report and Sarthak's bug log)
Conclusion & Future Scope
